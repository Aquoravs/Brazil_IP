### Current Pending Issues (MUST FIX BEFORE NEXT SESSION)

**Issue 1 — BLOCKER: Footnotes not visible on F-stat grid frames (frames 2–5)**

The `[c]` frame option combined with the current table (16 rows + headers at `\footnotesize`) fills
the frame height completely, pushing the `{\scriptsize ...}` note below the frame boundary.
The note is not cut — it simply doesn't render because the content overflows.

**Root cause:** `\footnotesize` table is too tall for `[c]` centered layout when the note must also fit.

**Recommended fix:**
Change frame option from `[c]` to `[t]` (top-aligned) on frames 2–5 only.
The table at current font size will sit at the top, leaving natural space below for the note.
The `[c]` option was added for aesthetics but causes the overflow.

Alternatively, revert table body font from `\footnotesize` back to `\scriptsize` (which fit on screen
as confirmed by screenshots). The note is already `\scriptsize` (upgraded from `\tiny` by the audit).
This was the state when the screenshots were taken and the notes showed correctly.

**Current state of the file:** `\footnotesize` table body + `[c]` frame → notes invisible.
**Target state:** Either `[t]` + `\footnotesize` OR `[c]` + `\scriptsize` table body.
User preference unknown — compile and show both options.

Frames to fix: lines 49, 105, 161, 217 (`\begin{frame}[c]{...}`)
Table font to fix: lines 51, 107, 163, 219 (`\footnotesize` → `\scriptsize` if reverting)

**Issue 2 — MEDIUM: `\sbox0` box register fragility in 26 appendix table files**

All 26 `\input{}`-ed tables use `\sbox0{...}` (LaTeX scratch register). Fix is in Script 51
(the R script that generates the table `.tex` files), not in `first_stage.tex` itself.
Change: `\sbox0{...}` → `\newsavebox{\mytablebox}` declared in preamble; `\savebox{\mytablebox}{...}` in each table.
This requires modifying Script 51's table output code.

**Issue 3 — LOW: `\multirow` + `\addlinespace` centering**

The `\addlinespace[5pt]` separator between Coalition and Party in each panel
is not counted by `\multirow` row-span calculations, so the Coalition/Party
labels appear slightly off-center vertically within their 4-row spans.
Fix: replace `\addlinespace[5pt]` with `\midrule` (or remove and use
`\cmidrule(lr){2-9}` to avoid underlining the Alignment column).
User previously preferred `\addlinespace` over `\cmidrule` — discuss before changing.
